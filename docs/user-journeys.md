# User Journeys

## Publish My Service

1. Node provider logs in.
2. Opens Publish My Service.
3. Completes service profile.
4. Adds docker image and container configuration.
5. Defines operation name, inputs, outputs, and supported DataReference protocols.
6. Runs control interface validation.
7. Adds documentation and example payloads.
8. Submits service for review.
9. Reviewer approves or requests changes.
10. Approved service appears in catalogue.

## Build a Solution

1. AI vendor opens catalogue.
2. Filters services by use case, provider, protocol, trust badge, or maturity.
3. Adds services to a workflow canvas.
4. Connects service outputs to downstream service inputs.
5. Configures workflow inputs and parameters.
6. Runs validation.
7. Submits workflow to orchestrator.
8. Monitors workflow and task status.
9. Reviews output DataReferences and result summaries.

## Govern Services

1. Reviewer opens approval queue.
2. Inspects submitted service metadata.
3. Checks control interface validation results.
4. Applies badges such as Validated, TEF-ready, or Experimental.
5. Approves, rejects, or requests changes.
